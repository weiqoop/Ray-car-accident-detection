
## 3. 詳細流程圖：Accident 車禍偵測

### 3.1 完整四階段 + 防洩漏設計

```
┌──────────────────────────────────────────────────────────────────┐
│                    ACCIDENT 車禍偵測流程                          │
└──────────────────────────────────────────────────────────────────┘

╔════════════════════════════════════════════════════════════════╗
║             Stage 1: Ray Data 影片級切分 (防洩漏)               ║
╠════════════════════════════════════════════════════════════════╣
║                                                                ║
║  輸入：TAD (Traffic Anomaly Dataset) frames/                  ║
║       ├─ normal/Normal_001.mp4/{0.jpg, 1.jpg, ...}           ║
║       │  (負類 250 支)                                        ║
║       └─ abnormal/01_Accident_001.mp4/{...}                  ║
║          (正類 110 支，其他異常類排除)                        ║
║                                                                ║
║  核心設計：影片級切分                                          ║
║    ┌────────────────────────────────────┐                    ║
║    │ ❌ 錯誤做法 (幀級隨機切)            │                    ║
║    │  影片 A → 幀 [0,1,2,...,9]          │                    ║
║    │           ↓ 隨機挑                 │                    ║
║    │  train: 幀 [0,2,4,6,8]  ← 同片!  │                    ║
║    │  test:  幀 [1,3,5,7,9]  ← 同片!  │                    ║
║    │  結果：模型「背」test，99% 虛假   │                    ║
║    └────────────────────────────────────┘                    ║
║                                                                ║
║    ┌────────────────────────────────────┐                    ║
║    │ ✅ 正確做法 (影片級分層切分)        │                    ║
║    │  影片 A ──→ train                  │                    ║
║    │  影片 B ──→ val                    │                    ║
║    │  影片 C ──→ test                   │                    ║
║    │  同支影片的幀「完全不跨」split      │                    ║
║    │  → test 影片在訓練時「完全看不見」 │                    ║
║    │  → test acc = 92% (誠實)           │                    ║
║    └────────────────────────────────────┘                    ║
║                                                                ║
║  處理步驟：                                                    ║
║    1. 列舉 360 支影片，分層隨機打亂                           ║
║    2. 按比例分配：                                            ║
║       • train: 175 支 (70%) ← 12,250 幀                      ║
║       • val:   54 支 (15%)  ← 3,240 幀                       ║
║       • test:  54 支 (15%)  ← 3,240 幀                       ║
║    3. 每支影片均勻抽 K 幀 (預設 60)                           ║
║    4. 解碼、縮放到 224×224、uint8                             ║
║    5. Ray Data 分散執行 (3 節點 CPU)                          ║
║                                                                ║
║  輸出：accident_tad_seq/                                      ║
║       ├─ train.npz {X, y, vid}                               ║
║       ├─ val.npz   {X, y, vid}                               ║
║       ├─ test.npz  {X, y, vid}                               ║
║       └─ videos.json {vid_id: {name, label}}                 ║
║                                                                ║
╚════════════════════════════════════════════════════════════════╝
                              ↓


╔════════════════════════════════════════════════════════════════╗
║          Stage 2: Ray Tune 超參搜尋（物件存儲共享）            ║
╠════════════════════════════════════════════════════════════════╣
║                                                                ║
║  輸入：accident_tad_seq/{train.npz, val.npz}                  ║
║                                                                ║
║  搜尋空間：                                                    ║
║    • backbone:        [mobilenet_v2, resnet18]               ║
║    • freeze:          [True, False]                          ║
║    • dropout:         [0.0, 0.4]                             ║
║    • aug:             [0.1, 0.3]                             ║
║    • lr:              [1e-4, 3e-3]                           ║
║    • weight_decay:    [1e-6, 1e-3]                           ║
║    • batch:           [32, 64]                               ║
║                                                                ║
║  關鍵技巧：物件存儲共享（Object Store）                        ║
║    ┌────────────────────────────────────┐                    ║
║    │ Driver 端（執行一次）              │                    ║
║    │ ┌──────────────────────────────┐  │                    ║
║    │ │ arrays = load_arrays(...)    │  │                    ║
║    │ │ ↓ 載入 2.3GB train array     │  │                    ║
║    │ │ ray.put(arrays)              │  │                    ║
║    │ │ ↓ 放進共享記憶體             │  │                    ║
║    │ └──────────────────────────────┘  │                    ║
║    │                                    │                    ║
║    │ 各 Trial 子程序（12 個）           │                    ║
║    │ ┌──────────────────────────────┐  │                    ║
║    │ │ tune.with_parameters(        │  │                    ║
║    │ │   trainable, data=arrays)    │  │                    ║
║    │ │ ↓ 零拷貝讀同一份 arrays       │  │                    ║
║    │ │ 訓練不會各複製 2.3GB          │  │                    ║
║    │ └──────────────────────────────┘  │                    ║
║    └────────────────────────────────────┘                    ║
║                                                                ║
║  演算法：ASHA (gpu_per_trial=0.5，單卡 2 trial 並行)          ║
║    Trial 1, 2, ..., 12 同時跑                                ║
║    ↓ 無進展自動砍掉                                          ║
║    ↓ 快速收斂                                                ║
║                                                                ║
║  輸出：最佳超參                                                ║
║    {backbone=mobilenet_v2, freeze=True, lr=7.5e-4,          ║
║     weight_decay=3.2e-5, f1=0.835}                          ║
║                                                                ║
╚════════════════════════════════════════════════════════════════╝
                              ↓


╔════════════════════════════════════════════════════════════════╗
║                  Stage 3: Ray Train 正式訓練                    ║
╠════════════════════════════════════════════════════════════════╣
║                                                                ║
║  輸入：accident_tad_seq/ + 最佳超參                            ║
║                                                                ║
║  模型架構：MobileNetV2 CNN 逐幀二分類                          ║
║    ┌────────────────────────────────────┐                    ║
║    │ 224×224 RGB 幀輸入                 │                    ║
║    │ ↓                                  │                    ║
║    │ ImageNet 預訓練 MobileNetV2        │                    ║
║    │ (或凍結 or 解凍主幹)               │                    ║
║    │ ↓                                  │                    ║
║    │ Global Average Pooling             │                    ║
║    │ ↓                                  │                    ║
║    │ [自寫] Linear(1280) → Sigmoid      │                    ║
║    │ ↓                                  │                    ║
║    │ 二分類邏輯值 (0~1)                 │                    ║
║    │ ↓                                  │                    ║
║    │ BCE Loss (正負樣本加權)             │                    ║
║    └────────────────────────────────────┘                    ║
║                                                                ║
║  訓練流程（TorchTrainer on GPU）：                            ║
║    • 15 epoch (可調)                                         ║
║    • 批次 64 (可調)                                          ║
║    • 增強：旋轉、翻轉、色彩抖動                              ║
║    • Dropout: 0.2~0.4 (防過擬合)                             ║
║    • 監控指標：val F1 (類別不平衡)                           ║
║                                                                ║
║  輸出：ray_results/accident_tad_final/accident_tad.pt         ║
║                                                                ║
╚════════════════════════════════════════════════════════════════╝
                              ↓


╔════════════════════════════════════════════════════════════════╗
║         Stage 4: 評估 (影片級聚合) + 儀表板推論               ║
╠════════════════════════════════════════════════════════════════╣
║                                                                ║
║  A. 評估 (eval_accident_tad.py)                               ║
║    ┌────────────────────────────────────┐                    ║
║    │ 1. 逐幀推論                        │                    ║
║    │    test 影片 (3,240 幀)             │                    ║
║    │    ↓ MobileNetV2 一批批推論        │                    ║
║    │    → 每幀分數 [0, 1]               │                    ║
║    │    → frame-level PR-AUC: 0.892    │                    ║
║    │                                    │                    ║
║    │ 2. 影片級聚合 (關鍵)                │                    ║
║    │    影片 1: [0.1, 0.15, 0.2, ...]  │                    ║
║    │    ↓ 取第 90 百分位                │                    ║
║    │    → 影片分數 = 0.75               │                    ║
║    │                                    │                    ║
║    │    整個 test set (54 影片)          │                    ║
║    │    ↓ 計算 ROC-AUC / PR-AUC / F1   │                    ║
║    │    → 混淆矩陣 (TP/FP/FN/TN)       │                    ║
║    │                                    │                    ║
║    │ 結果：                              │                    ║
║    │   ROC-AUC: 0.960  ← 主指標         │                    ║
║    │   PR-AUC:  0.901                   │                    ║
║    │   F1:      0.909                   │                    ║
║    │   混淆矩陣：TP15 FP1 FN2 TN36      │                    ║
║    │   (17支事故 88% 抓到, 97% 不誤報) │                    ║
║    └────────────────────────────────────┘                    ║
║                                                                ║
║  B. Ray Serve 推論展示                                        ║
║    ┌────────────────────────────────────┐                    ║
║    │ 1. 自動選片邏輯                    │                    ║
║    │    • 優先選「有明顯上升趨勢」      │                    ║
║    │    • ~3 事故片 + ~3 正常片         │                    ║
║    │    → 視覺效果：模型逐漸升信心     │                    ║
║    │                                    │                    ║
║    │ 2. 儀表板面板 (右下角)              │                    ║
║    │    ┌──────────────────────────┐   │                    ║
║    │    │ 車禍偵測面板              │   │                    ║
║    │    │ 影片 1/6, 幀 3/60         │   │                    ║
║    │    │ [幀預覽]                  │   │                    ║
║    │    │ 分數: 0.75                │   │                    ║
║    │    │ ⚠️ 偵測到車禍事件        │   │                    ║
║    │    └──────────────────────────┘   │                    ║
║    │                                    │                    ║
║    │ 3. 刷新頻率                        │                    ║
║    │    • 2 秒換下一幀 (與 Freeway 同步) │                    ║
║    │    • GPU 推論 < 10ms               │                    ║
║    │                                    │                    ║
║    │ 輸出：http://localhost:8000        │                    ║
║    │       右下角「車禍偵測」面板         │                    ║
║    └────────────────────────────────────┘                    ║
║                                                                ║
╚════════════════════════════════════════════════════════════════╝
```

## 4. 兩條流程並列對比

```
│                                                                │
│           FREEWAY (車流)              ACCIDENT (車禍)           │
│           ─────────────              ─────────────             │
│                                                                │
│  輸入      高公局 CCTV                TAD frames (360 支)       │
│           標註資料                   + 標籤                    │
│                                                                │
│  Stage 1  Ray Data                  Ray Data                 │
│           鏡頭級切分                 影片級切分               │
│           (防同鏡頭洩漏)             (防同片洩漏)             │
│           ↓                          ↓                        │
│           freeway_prepared/          accident_tad_seq/       │
│           (images + labels)          (train/val/test.npz)    │
│                                                                │
│  Stage 2  Ray Tune (ASHA)            Ray Tune (ASHA)          │
│           搜: lr0, weight_decay,     搜: backbone, freeze,   │
│               aug 參數               dropout, aug, lr, wd    │
│           → 單卡並行 trial           → 單卡 2 trial 並行      │
│                                      (物件存儲共享)           │
│                                                                │
│  Stage 3  Ray Train                 Ray Train                │
│           yolo11s                   MobileNetV2 CNN          │
│           (檢測)                     (二分類)                 │

│                                                                │
│  評估      mAP50 / mAP50-95          影片級聚合               │
│           (即時)                     ROC-AUC: 0.96           │
│                                      PR-AUC: 0.90            │
│                                      F1: 0.91                │
│                                                                │
│  Stage 4  Ray Serve                 Ray Serve                │
│           5 路 CCTV 即時             TAD test 影片 demo      │
│           + YOLO 推論               + MobileNetV2 逐幀      │
│           → 左側面板                → 右下角面板             │
│           (2s 刷新)                 (2s 換幀)               │
│                                                                │
│  輸出      http://localhost:8000 (統一儀表板)                 │
```
